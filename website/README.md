# Spawnly documentation site

A [Starlight](https://starlight.astro.build/) (Astro) site that publishes the
Markdown in [`../docs`](../docs) as a searchable, navigable documentation
website.

## Run it

```bash
cd website
npm install        # first time only
npm run dev        # live-reload dev server at http://localhost:4321
npm run build      # production build into website/dist/
npm run preview    # serve the production build locally
```

## How content is sourced

The canonical docs stay in [`../docs`](../docs) — this site does **not** copy
them. [`src/content.config.ts`](src/content.config.ts) globs `../docs/**/*.md`
into Starlight's `docs` collection, so editing a file in `docs/` is all you need;
the site picks it up on the next build.

Two small remark plugins adapt the in-repo Markdown for the web without changing
the source files:

- [`plugins/remark-strip-first-h1.mjs`](plugins/remark-strip-first-h1.mjs) —
  drops the leading `# Title` at render time (the files keep it so they read well
  on GitHub; Starlight renders the title from frontmatter, so this avoids a
  duplicate heading).
- [`plugins/remark-rewrite-repo-links.mjs`](plugins/remark-rewrite-repo-links.mjs)
  — rewrites each relative link: doc → doc becomes an internal site route, and
  doc → source (e.g. `../../internal/foo.go#L10`) becomes a GitHub URL
  (`blob` for files, `tree` for directories), preserving `#` anchors.

## Adding a page

1. Add a Markdown file under [`../docs`](../docs) with frontmatter:
   ```yaml
   ---
   title: My Page
   description: One-line summary.
   ---
   ```
2. Add it to the `sidebar` in [`astro.config.mjs`](astro.config.mjs) as a
   `{ slug: '...' }` entry in the relevant group. (The sidebar is listed
   explicitly rather than autogenerated, because the docs are loaded via
   `glob()` from `../docs` and Starlight's `autogenerate` only scans the default
   `src/content/docs` path.)

## Deployment

Currently configured for **local build only** (served at the root, no `base`).
To publish on a GitHub Pages project site later, set `site` and
`base: '/Spawnly/'` in [`astro.config.mjs`](astro.config.mjs) and add a Pages
deploy workflow.
